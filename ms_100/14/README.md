# problem
```
题目:输入一个已经按升序排序过的数组和一个数字,
在数组中查找两个数,使得它们的和正好是输入的那个数字。
要求时间复杂度是 O(n)。如果有多对数字的和等于输入的数字,输出任意一对即可。
例如输入数组 1、2、4、7、11、15 和数字 15。由于 4+11=15,因此输出 4 和 11。
```

# 分析
```
一开始想的慢方法：
2重循环。。。 这个没有利用数组递增的特点。。。


o(n)的方法：
使用两个指针，p,q 分别指向第一个和最后一个元素。
当 *p + *q == dest的时候，说明找到了(并且p<q)

当p q的和sum (也是利用了有序的特点)
	sum>dest 需要减小sum，所以指针q要前移
	sum<dest 需要增加sum，所以指针p要后移

类似一个两边夹的过程。
```

# 总结
```
判断两个数的和是否等于某个数，第一感觉是：使用两个指针。
```
